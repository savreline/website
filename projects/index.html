Chevrotain: a distributed key-value store

An individual course project for the distributed systems abstractions graduate course. Designed and implemented three different variants of a replicated, multi-primary key-value store that achieves eventual consistency via conflict-free replicated data types (CRDTs).  GitHub

Compared and contrasted performance of the different variants by subjecting them to various loads while deployed on ten geographically distributed Azure VMs. One of the variants achieved a sub-second latency under a throughput of 10 000 operations per second and out-performed MongoDB's built-in replication service (which follows the primary-backup model) under this load.  Technical Report

While working on the project, contributed a new feature and repaired a bug in GoVector, a UBC distributed system research group's open source Golang library.  GitHub

Golang and MongoDB. 2020.

"What year was that?" trivia game

A single page web application where users earn points based on how closely they are able to guess the years in which various world history events took place. Correctness of a user's guess relative to other user's guesses is shown on a D3.js chart. Communication with the database is via server-side API.  GitHub

Deployed project on an AWS EC2 instance running Apache2 web server and on MongoDB Atlas. Deployed version can be viewed at:  savreline.com

Course notes on algorithms

A comprehensive set of course notes developed for students taking UBC's CPSC 320 course entitled "Intermediate Algorithm Design and Analysis". Topics include stable matching problem, graph algorithms, greedy algorithms, divide and conquer, dynamic programming and NP-completeness.

Notes have been referenced by several cohorts of students taking CPSC 320 and were found to be a very effective study supplement.  PDF
